Enterprise Entity Data Validation

Overview

A six-table relational schema that validates company and entity records from multiple data sources and shows exactly where the content can't be trusted, a hands-on model of real entity / reference-data quality work.

Key features

  • Flags duplicates and stale records using joins, subqueries, CASE logic, and aggregate functions.
  • Detects missing identifiers (LEI, DUNS, Tax ID, VAT) across the entity population.
  • Produces seven data-quality reports: PASS/FAIL/WARNING validation, freshness, and coverage by country and industry.
dq_status.sqlexcerpt
-- classify each entity on identifier & freshness
SELECT e.legal_name, e.country,
       CASE WHEN i.lei IS NULL              THEN 'FAIL'
            WHEN e.updated_at < '2025-01-01' THEN 'WARN'
            ELSE 'PASS' END AS dq_status
FROM entities e
LEFT JOIN identifiers i ON i.entity_id = e.id;

AI Content & Knowledge Management

Overview

A normalized six-table database that catalogues reusable content assets by type, vertical, and owner, logs usage across projects, and tracks review history, so the content library stays findable, fresh, and genuinely reusable.

Key features

  • Catalogues assets by type, vertical, and owner; logs usage across projects.
  • Tracks review history to keep the library accurate and current.
  • Wrote queries joining usage and review logs to report on content usage, freshness, and reusability.
content_reusability.sqlexcerpt
-- most-reused assets, with last review date
SELECT a.title, a.vertical, COUNT(u.id) AS uses,
       MAX(r.reviewed_at) AS last_review
FROM assets a
JOIN usage_log u ON u.asset_id = a.id
GROUP BY a.title, a.vertical
HAVING COUNT(u.id) > 5;
